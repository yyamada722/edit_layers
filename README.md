# Edit Layers

3ds Max の Edit Poly のような、レイヤーベースの非破壊メッシュ編集スタックを Blender に追加するアドオン。
トポロジ変更(押し出し・ベベル・サブディバイド・削除など)を含む編集をレイヤーとして記録し、
後から個別に無効化・並べ替え・再編集できる。さらに任意のレイヤーから**ブランチ**を分岐させ、
編集パターンを切り替え・比較できる。

## ヘルプ

画像付きのヘルプを [docs/index.html](docs/index.html) に同梱している
(パネル下部の **?** ボタンからも開ける)。スクリーンショットは
`scratchpad` の撮影スクリプトで Blender 実機から自動生成したもの。

## 使い方

1. `View3D > Sidebar > Edit Layers` パネルで **スタックを初期化**
2. **新規レイヤーを記録** → 通常の Edit Mode で自由に編集 → **コミット**
3. レイヤーリストで表示切替 (目アイコン) / 並べ替え / 削除 / **選択レイヤーを再編集**
   - リスト横の **▼ メニュー** から「直前のレイヤーと統合」「ここまでをベースに確定」
     (部分ベイク) でスタックを整理できる
   - ヘッダーの **オーバーレイアイコン** で、選択レイヤーが影響した頂点を
     ビューポートにハイライト表示 (橙: 移動 / 緑: 生成)
4. 完成したら **ベイク** でスタックを確定・削除

専用の編集オペレータは不要。記録中は Blender 標準の編集ツールがすべて使える。

### スカルプトモード

「記録 (スカルプト)」でスカルプトモードに入って記録できる。差分エンジンはモード非依存
なので、記録中に編集/スカルプトを自由に行き来してもよく、コミット時のメッシュとの
差分がまとめて 1 レイヤーになる。

- 通常のブラシストローク (位置変更) は `moved` delta として軽量に記録される
- Dyntopo・リメッシュはトポロジが全面的に変わるため「ほぼ全削除 + 全生成」の
  重いレイヤーになる (壊れはしないが差分の利点は失われる)
- マルチレゾモディファイアの変位はベースメッシュ外のデータなので記録されない

### シェイプキーとの関係 (排他・ロックあり)

シェイプキーは頂点インデックスに直接依存するデータで、本アドオンの再構築
(bmesh -> mesh 書き込み) はキーのデータを Basis で上書きしてしまう。このため
シェイプキーとの併用は**排他**とし、双方向にロックしている:

- キーのあるメッシュでは**スタックを初期化できない** (先にキーを適用/削除する)
- スタック運用中に**シェイプキーを追加すると即座に自動で取り消される**
  (depsgraph ハンドラで検知。追加直後なので失われるデータはない)。
  パネルに「ブロックしました」の通知が表示される
- 例外: キーとスタックが共存した状態で保存された古いファイルを開いた場合は、
  既存キーを勝手に削除せず、警告 + 再構築系操作のブロックに留める
  (レイヤーの表示切替も保留され、キーもメッシュも壊れない)
- **「スタックを破棄 (現状を確定)」**で、現在のメッシュとキーを保持したまま
  スタックだけを外して通常のワークフローに戻れる

推奨ワークフロー: モデリング (Edit Layers) を終えてベイクした後にシェイプキーを
作成する。

### 記録し忘れの救済

記録を開始せずに Edit Mode で直接モデリングしてしまっても救済できる:

- 未記録の編集は自動検出され、パネルに**「未記録の編集があります」**の警告と
  **「レイヤーとして取り込む」/「破棄して再構築」**ボタンが表示される
- 「取り込む」は最後の再構築状態を内部で再現し、現在のメッシュとの差分を
  遡って新規レイヤー化する (通常のコミットと同じ品質で残る)
- 未記録編集がある間、それを消してしまう操作 (記録開始・並べ替え・削除・
  ベイク等) はブロックされ、レイヤーの表示切替は保留される
- 記録せずに Edit Mode に入った時点でもパネルに警告が出る

注意: 検出はメッシュの指紋 (要素数と座標和) の比較なので、ファイルを開き直した
直後は再構築履歴がなく検出できない。その場合も「取り込む」自体は現在の構成から
近似して実行できる。

### ブランチ

- レイヤーを選択して **[+]** (ここからブランチ) → そのレイヤーを分岐点とする新ブランチが
  作られ、以降のコミットは新ブランチにだけ積まれる
- ブランチリストのクリックで即座に切り替え (メッシュが再構築される)
- **比較** で他ブランチの結果をオブジェクト複製として横に並べる / **解除** で片付ける
  (比較複製をユーザーがさらに複製したものは「解除」で削除されず、通常オブジェクトとして残る)
- 各レイヤーの行頭 (1 枠): このブランチ専用のレイヤーにはブランチ色のチップ、
  共有レイヤーは空白 (下に行くほど「自分の色の帯」になる)
- 分岐点のレイヤーには「← ブランチ名」のバッジが色チップ付きで表示される
- 分岐点より上流の共有レイヤーを再編集すると**全ブランチに波及**する
- 共有レイヤーの削除・共有レイヤーをまたぐ並べ替えは安全のためブロックされる
- ブランチ削除時は、そのブランチ専用のレイヤーだけが一緒に削除される
- ベイクは**アクティブブランチの結果**を確定する

## 仕組み

- **永続頂点 ID**: 初期化時に全頂点へ INT 属性 `el_id` で不変 ID を付与する (0 = 未割り当て)。
  レイヤーの記録・再生はすべて頂点インデックスではなくこの ID を参照するため、
  上流レイヤーの変更でインデックスがずれても参照が壊れない。
- **レイヤー = トポロジ差分**: 編集セッションの前後スナップショットを ID 基準で比較し、
  以下を JSON としてレイヤーに保存する:
  - `moved` — 既存頂点の移動 (delta)。上流で頂点が動いても相対的に追従する
  - `new_verts` + `anchors` — 新規頂点。近傍の既存頂点 3 点をアンカーとして
    「アンカー重心 + オフセット」で保存するため、上流の変形に追従する
    (アンカーが失われた場合は絶対座標にフォールバック)
  - `face_attrs` / `edge_attrs` / `vert_attrs` — マテリアルインデックス・スムーズ・
    シーム・シャープ・クリース・ベベルウェイトの変更
  - `deleted_verts` / `deleted_edges` / `deleted_faces` — 削除 (連鎖削除されるものは記録しない)
  - `new_edges` / `new_faces` — 新規トポロジ (頂点 ID 参照)
- **レイヤーツリー**: レイヤーは線形リストではなくツリー (各レイヤーが親レイヤーの UID を参照)。
  ブランチは末端レイヤー (head) へのポインタで、アクティブブランチ = head から根まで
  遡ったパス。分岐点より上流のレイヤーは実体が共有されるため、コピー方式と違って
  上流の修正が全ブランチに自動で波及する。
- **再構築**: 初期化時に退避したベースメッシュのコピーへ、アクティブパスの差分を bmesh で順に適用する。
- **ID の正規化**: subdivide 等で INT 属性が複製されて ID が重複した場合、
  編集前の位置に最も近い 1 頂点だけが ID を保持し、残りは新規頂点として採番し直す。
- **壊れた参照**: 上流の変更で参照先 ID が消えた場合、生成・移動はスキップして警告を
  パネルに表示する。削除系は対象がなければ黙ってスキップする (実害がないため)。

## 既知の制限

- 新規頂点はアンカー相対で追従するが、追従するのは平行移動のみ
  (上流の回転・スケールにはオフセットが回転しない)。
- **UV と頂点カラーは記録しない** (マテリアルインデックス・スムーズ・シーム・
  シャープ・クリース・ベベルウェイトは v0.8 から記録される)。
- 記録セッションの編集前スナップショットはメモリ上にのみ保持されるため、
  **記録中に Blender を終了すると記録は失われる** (レイヤー自体は .blend に保存される)。
- 未記録編集の検出はセッション内の再構築履歴に依存する (ファイルを開き直した直後の
  未記録編集は自動検出されない。手動で「取り込む」ことは可能)。
- 同一頂点集合で構成される二重面は区別できない。

## 言語

UI は英語がソースで、日本語訳を内蔵している。Blender の言語設定
(プリファレンス > インターフェイス > 翻訳) が日本語ならパネル・ボタン・
メッセージが自動で日本語表示になる。

## データの保存場所

- レイヤースタック: `Object.edit_layers` (PropertyGroup、.blend に保存)
- ベースメッシュ: `<メッシュ名>_el_base` (fake user 付き Mesh データブロック)
- 永続 ID: メッシュの `el_id` INT 属性 (POINT ドメイン)
